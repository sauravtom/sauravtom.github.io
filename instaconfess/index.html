code




Its a bueatiful day and my weekend hack project is finally done, and it took far less time to completion than anticipated, a evening full of laziness and parallell skype talking and ofcourse reddit. (I need to mend my ways).

No project was ever complete without a boring introduction, so here goes.

The confession pages trend is really picking up in colleges, and DTU,my college is no different.One of the confession pages got 3k likes within few months, and the amount of user engagement is just marvellous.I won't be surprised if it got anything near 50k monthly views. But I noticed something strange with the confession pages.Behind almost all of these pages what really used to be hapenning was that a neckbeard would made a facebook confession page and would make a google form and ask users to post their confessions on that google form, he then would open up google drive every once in a while and start posting the submissions himself.

"All your confessions are belong to us" ~Big Brother

That is barbarian, I thought that there must be a better way to do this, and there indeed was.

instaconfession.appspot.com So I made a facebook page and the accompanying bot to do the same. You can find the source code on github here. github.com/sauravtom/instaconfess

This is actually the source code for the whole instaconfess website, which includes a form, a captcha and the ofcourse the autoposter.

Writing the code wasn't the big deal, what was big deal though was configuring the access_token bullshit of facebook pages.

Seems like facebook really doesn't want programmers to interact with their API if ther are using python.

No official python facebook SDK, seriously facebook ? You can do better.

And that is only the tip of the ideberg, configuring the access_tokes and redirect_uri is a developer nightmare. As always StackOverflow came to the rescue.

The trick here is to, first make your page and everything as usual, and then going to the Graph explorer tool and generating the access_toke, this token however is only valid for 2 hours.

To extend the access_token to 60 days you need to follow these instructions,

https://graph.facebook.com/oauth/access_token?
grant_type=fb_exchange_token&
client_id=APP_ID& client_secret=APP_SECRET& fb_exchange_token=SHORT_LIVED_ACCESS_TOKEN

Add the values in the fields accordingly, and then you will get a response output of your access_token extended to 60 days.

There is a way you can extend this access token even beyond 60 days, I haven't been able to do so yet.
