Mapping multiple domains on the same IP on different ports


Published on 08 PM Sunday 27 July 2014

code




We can setup different domains on the same server on different ports, using apache2 and uWSGI. uWSGI is a deployment option on servers like nginx, lighttpd, apache2 to serve your app (flask,django,hell even php).

For a primer about dealing with multiple domains on your VPS, give this verbose blog post a read, written by my hacker friend championswimmer.

So here goes my little how to.

First make a socket connection using uWSGI

[uwsgi]
master = true
socket = :3031
wsgi-file = /path/to/flask/file.py
module = file_name
callable = app
daemonize = /var/log/uwsgi.log
pidfile = /var/run/uwsgi.pid
process   = 4

In my case I am using port 3031, to find out which ports you can use, see this answer.

Now lets configure apache2, for that we need to edit the file

/etc/apache2/sites-available/<your-domain>

which is the cofiguration file for your website. Again check out championswimmer's blogpost, if you need more clarity.

We need to basically add a location block in that file, as shown below.So that uWSGI and Apache2 can communicate with one another.

<VirtualHost *:80>

    ...

    <Location />
        Options FollowSymLinks Indexes
        SetHandler uwsgi-handler
        uWSGISocket 127.0.0.1:3031
    </Location>

    ...

</VirtualHost>

After that is done, we need to install some modules required by apache2 to connect to uWSGI

sudo apt-get install apache2 libapache2-mod-uwsgi
sudo apt-get install uwsgi uwsgi-plugin-python uwsgi-plugin-cgi

After, that just restart spache2 server

sudo service apache2 restart

and run your uWSGI config file

uwsgi --ini /path/to/config/file.ini

And thats all you need to do.

If you still are having any doubts leave comments below.
