Scaling flask applications with NGINX and uWSGI


Published on 07 AM Monday 28 April 2014

code




In this tutorial I am going to describe the process of scaling your flask application to 1000s requests per seconds and be able to handle huge surge in web-traffic.

Setting up a Load Balancer on NGINX

Nginx is a modern webserver which is really catching up to Apache in popularity, in terms of performance it has already been pegged higher than apache.

A load balancer is basically a webserver which distribute the incoming requests amongst several backend servers which are invisible to the the user making the API request.

There are several advantages of using a load balancer + You are protected against DDOS attacks on your data servers since these are invisible. + You can serve more requests simultaneously. + You can take care of server downtimes more gracefully. + Fault tolerance and heightened stability.

So lets setup a basic Load Balancer on nginx,we will be setting up a Round Robin Load Balancer.The Round Robin algorithm for load balancing sends visitors to one of a set of IPs. At its most basic level Round Robin, which is fairly easy to implement, distributes server load without implementing considering more nuanced factors like server response time and the visitors’ geographic region.

First install nginx

sudo apt-get install nginx

Next, we need to setup the Upstream module in nginx default site configuration for our loadbalancer.

vim /etc/nginx/sites-available/default

Its gonna look something like this after you edit it,

upstream backend  {
  server 173.255.XXX.XXX:5000;
  server 198.58.XXX.XXX:8000;
  server 198.58.XXX.XXX:8000;
}

server {
    #listen   80; ## listen for ipv4; this line is default and implied
    #listen   [::]:80 default ipv6only=on; ## listen for ipv6

    root /usr/share/nginx/www;
    index index.html index.htm;

    server_name <your-server-ip>;

    location / {
        proxy_pass  http://backend;
        # First attempt to serve request as file, then
        # as directory, then fall back to index.html
        try_files $uri $uri/ /index.html;
        # Uncomment to enable naxsi on this location
        # include /etc/nginx/naxsi.rules
    }
}

So as you can see, we have specified the 3 backend server we want to balance to load to in the upstream stanza.After entering your server ip in the server_name line, you just do

sudo service nginx restart

and then go to your server ip on your webbrowser and you will find that it starts routing requests.For testing purposes I would recommend you websites like google.com, quora.com instead of backend servers.

Now that we have setup to nginx load balancer we need to host our flask app at the backend servers we specified in the last step.

After all is done this is what the whole architecture will look like.

A : nginx load balancer webserver

B : Data Banks, Static file hosting web servers

C : Client App

  1. Client makes a request to the central server (A) , the server then does load balancing, and finds the address of the least used Data Bank.(to be returned to client in next step)

  2. The Central Redirection Server, then returns the address in previous step to the client.

  3. The client then eiter makes a second request to the Data Bank server it has been assigned, and retrieves the data specified by the user or the url it first made request to redirects it to one of the backend servers, just like we are about to do.

If you are going to be using flask for production then it is strongly recommended that you use a webserver like gunicorn, uWSGI instead to flask's built in web server.

In this tutorial we will be using uWSGI

First install uWSGI

sudo apt-get install uWSGI

Next setup a basic app with uWSGI and see if everything is working fine or not, 

uwsgi --http :8000 --wsgi-file test.py

Where test.py is your flask app.After running th above command you should be seeing your app server at localhost:8000.

You can make config files for uWSGI if your parameters are too many like this I made.

[uwsgi]
uid = www-data
gid = www-data
master = true
socket    = :5000 
#plugin    = python
wsgi-file = /path/to/your/python/flask/app.py
module = name_of_your_file
callable = app
daemonize = /var/log/uwsgi.log
pidfile = /var/run/uwsgi.pid
process   = 4

Remember, we are doing this is one of the backend servers, this IS NOT the central server.
