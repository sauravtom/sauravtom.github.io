Hacking productivity with cron jobs


Published on 09 AM Friday 25 April 2014

general




When I first learned about the idea of adding sites to be blocked in /etc/hosts, I was delighted, mainly because I like such ideas which tend to enhance productivity. I began using it exhaustively, but soon enough I realised that even though it provided a decent way to get away from nagging sites like facebook and twitter, this method wasn't very successfull in time management, I habituated myself to edit /etc/hosts file with ease and hence lowering the barrier to attend to the itch.

The success of any good productivity hack is directly propotional to the barrier it provides to motivate you to not to attend to the itch, in this case the barrier was as low as editing a file in vim.

Not anymore, I have written a tiny python script with changes my /etc/hosts file every 1 minutes or so, with a previously specified template, thanks to cronjobs in linux.

So whenever I feel like checking my facebook or twitter, I still edit my /etc/hosts file, but I only get a 1 minute window to do all that I can before it gets restored again to pre-specified template. After that facebook.com gets re-added to my bloked sites list and I will again have to edit /etc/hosts to continue browsing facebook, which I seldom do. See, in this case the barrier is not only to edit the /etc/hosts file but also to do it repeatedly after every minute, which in my opinion is huge.

It has been very effective for me ever since I started using it, I have begun to use facebook exclusively as a messenger (which I use from my mobile) to connect with college friends, and check college groups feed (for bunks and notices) every once in a while on desktop. Needless to say a lot of time is saved, I no longer pursue meaningless pursuits like checking out profiles of my highschool friends and drooling in nostalgia on a boring saturday noon.

Same is with twitter.

You can find the aforementioned script on my github https://github.com/sauravtom/productivity

P.S It uses cronjobs so it only supports linux as of now, but support for more OSes can be added in the future. Pull requests are welcome.
